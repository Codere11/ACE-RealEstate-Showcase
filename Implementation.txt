==================================================
3. ROLE MODEL
==================================================

There are 3 roles:

1. PLATFORM_ADMIN
   This is us / internal operator.
   Can:
   - create organizations
   - create first org admin users
   - manage onboarding
   - see platform-wide data

2. ORG_ADMIN
   This is the client organization admin.
   Can:
   - manage only their own organization
   - manage org settings
   - later manage surveys / qualifiers / branding / payments within their own org

3. MANAGER
   Operational user inside one organization.
   Can:
   - later review leads
   - later handle live help / operational workflows
   - does not get platform-wide access

Login model:
- one login page: /login
- users authenticate there
- what they see after login depends on role

Important distinction:
- platform admin dashboard is not the same as the organization dashboard
- client organizations must not see platform-wide onboarding/control tools
